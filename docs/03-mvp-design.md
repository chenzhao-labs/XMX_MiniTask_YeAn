# 03｜MVP 设计说明

> 本文档阐述：「夜安」项目设计 MVP

## 1. MVP 目标

本项目的 MVP 目标不是直接做出可量产硬件，而是验证三个关键问题：

| 编号 | 问题 |
|---|---|
| Q1 | 用户是否能理解“AI远程陪护 Agent”的产品价值 |
| Q2 | 基于模拟行为数据，Agent 是否能生成有用的状态摘要 |
| Q3 | 子女是否愿意为“父母状态可见”而非单纯硬件报警付费 |

因此，MVP 重点放在：

- 子女端产品体验；
- Agent 工作流；
- 模拟硬件数据到 AI 输出的完整链路；
- 可运行 Demo 系统。

---

## 2. MVP 范围

### 2.1 当前实现

本 MVP 已实现：

- 模拟床脚传感器数据；
- 行为事件结构化；
- 4 个大模型 Agent；
- DashScope API 调用；
- 子女端小程序风格界面；
- AI 周报生成；
- 自然语言问答；
- 紧急事件分级决策演示。

### 2.2 当前未实现

本 MVP 尚未实现：

- 真实床脚硬件采集；
- 真实家庭长期部署；
- 医学级准确性验证；
- 批量硬件生产；
- 老人端 App 或设备端交互。

这是有意为之，因为本阶段验证重点是：

> Agent 是否能把行为数据转化为用户觉得有价值的陪护信息。
